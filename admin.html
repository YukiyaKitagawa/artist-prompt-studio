絵師ペルソナ管理

index.htmlで選択できる絵師ペルソナを管理します。胸像画像を選択して保存すると、artist_persona_imagesへアップロードされます。 既存画像のまま絵師カテゴリを変更した場合は、Storage内の画像配置も新しいカテゴリフォルダへ同期できます。

採用画像から、再利用できる絵師ペルソナを作成したいときの依頼文をコピーできます。最小構成版は、画風再現に必要な5項目へ絞ってsystem_instruction化します。

胸像画像

PNG / JPEG / WebP に対応しています。画像を選択して保存すると、Storageへアップロード後に image_url へ自動反映されます。

Preview

新しい画像を選択した場合は、保存時にアップロードされます。 画像を選択しない場合は、下の image_url の値を使用し、カテゴリ変更時にStorage画像配置を同期できます。

artist_persona_images バケット内のパスです。Private Storageのまま使用できます。

絵師カテゴリ管理

絵師ペルソナを「所属AI」「用途」「画風系統」などで分類するカテゴリを管理します。

Storageの保存先フォルダ名として自動入力される場合があるため、半角英数字を推奨します。

使用中の絵師カテゴリを削除すると、DB側では絵師ペルソナの category_id が null になります。 画像配置まで移動したい場合は、削除ではなく絵師ペルソナ編集画面でカテゴリを変更して保存してください。

キャラクターカテゴリ管理

キャラクターを分類するカテゴリを管理します。使用中のカテゴリは安全のため削除できません。

Storageの保存先フォルダ名として自動入力される場合があるため、半角英数字を推奨します。

カテゴリを削除する場合は、先にそのカテゴリを使用しているキャラクターのカテゴリを変更するか、キャラクターを削除してください。 画像配置まで移動したい場合は、キャラクター編集画面でカテゴリを変更して保存してください。

キャラクター管理

画像ファイルを選択して保存すると、画像アップロードとキャラクターレコード保存をまとめて行います。 既存画像のままカテゴリを変更した場合は、Storage内の画像配置も新しいカテゴリフォルダへ同期できます。

画像

PNG / JPEG / WebP に対応しています。画像を選択して保存すると、Storageへアップロード後に image_url へ自動反映されます。

Preview

新しい画像を選択した場合は、保存時にアップロードされます。 画像を選択しない場合は、下の image_url の値を使用し、カテゴリ変更時にStorage画像配置を同期できます。

既存画像を使う場合は、ここに character_images バケット内のパスを直接入力できます。

Storage画像管理

character_images と artist_persona_images の画像を確認・削除できます。

空欄の場合はバケット直下を表示します。カテゴリ名や所属AI名のフォルダを指定するのがおすすめです。

注意

Storage画像を削除しても、DB側の image_url は自動では消えません。 不要画像かどうか確認してから削除してください。
